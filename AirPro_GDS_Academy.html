Module 01 · GDS পরিচিতি

Global Distribution System (GDS) — সম্পূর্ণ পরিচয়

GDS কী, কীভাবে কাজ করে, বিশ্বের প্রধান GDS platform গুলো এবং বাংলাদেশের travel industry তে এর ভূমিকা।

🌐
GDS কী?

GDS (Global Distribution System) হলো একটি কেন্দ্রীয় computerized network platform যা airlines, hotels, car rentals এবং travel agencies এর মধ্যে real-time information আদান-প্রদান করে। এটি মূলত airlines এর inventory (আসন, ভাড়া) travel agents কে access দেওয়ার জন্য তৈরি।

সহজ ভাষায়: আপনি যখন Biman Bangladesh Airlines এর ticket কাটতে travel agency তে যান, agent GDS terminal এ command দিয়ে সরাসরি airline এর seat inventory দেখে এবং book করে।

GDS এর কাজ করার পদ্ধতি

1
Airlines CRS → GDS
প্রতিটি airline এর নিজস্ব Computer Reservation System (CRS) থাকে। Biman এর SITA, Emirates এর AOS ইত্যাদি। এই CRS গুলো GDS এর সাথে connected থাকে।
2
GDS → Travel Agencies
GDS এই airlines এর seat availability, fares, schedules একত্রিত করে travel agencies কে দেখায় একটি single interface এ।
3
Booking → Confirmation
Travel agent booking করলে GDS airlines এর CRS কে instantly update করে, passenger কে PNR (booking reference) দেয়।

Module 03 · Amadeus Basic Commands

Amadeus Cryptic Commands — মূল ভিত্তি

Amadeus GDS এ cryptic (command-based) interface ব্যবহার করার পদ্ধতি এবং সব ধরনের basic commands।

📌
Amadeus Command Structure

Amadeus Cryptic এ প্রতিটি command একটি নির্দিষ্ট format অনুসরণ করে। Commands সাধারণত uppercase এ type করা হয় এবং Enter বা ; দিয়ে execute হয়।

Amadeus Cryptic Terminal — Command Structure
-- Command Format: [Action] [Subject] [Modifier]
AN = Availability [Action] + [N=Next]
SS = Sell Segment
NM = Name [passenger]
AP = Address/Phone
TL = Ticketing Limit
ER = End & Retrieve
ET = End Transaction
IG = Ignore (cancel changes)
-- সহজ নিয়ম: প্রথম ১-২ অক্ষর = কী করবেন, বাকিটা = বিস্তারিত তথ্য

Module 06 · PNR তৈরি

PNR তৈরি — সম্পূর্ণ Step-by-Step Guide

Passenger Name Record (PNR) কী, কীভাবে তৈরি করতে হয়, mandatory elements কী কী এবং সম্পূর্ণ booking workflow।

📋
PNR কী?

PNR (Passenger Name Record) হলো GDS এ একটি booking record যা একজন বা একদল যাত্রীর সমস্ত travel information ধারণ করে। প্রতিটি PNR এর একটি unique 6-character alphanumeric code থাকে যাকে Record Locator বলে। যেমন: XYZ123

PNR এর Mandatory Elements (5টি অবশ্যক)

1
Name Element (NM) — যাত্রীর নাম
কমপক্ষে একজন যাত্রীর নাম থাকতে হবে। Format: Surname/Firstname Title
2
Itinerary Element — flight segment
কমপক্ষে একটি flight বা segment book থাকতে হবে।
3
Contact Element (AP) — যোগাযোগ
Phone বা email — emergency contact করার জন্য অবশ্যক।
4
Ticketing Element (TK/TL) — টিকেটিং সময়সীমা
Ticket করার শেষ তারিখ বা TKOK (OK to ticket immediately)।
5
Received By (RF) — কে বুক করল
Travel agent বা passenger এর নাম — কে booking করেছে।

E-Ticket Output — TWD Command
> TWD
ELECTRONIC TICKET RECORD TKT: 176-2345678901 ISSUED: 15NOV25 NAME: AHMED/KARIM MR ITIN: EK583/Y 15NOV DACDXB HK FARE: BDT 6800 TAX: BDT 3210 TOTAL: BDT 10010 FOP : CASH FBAS: KLBDAC VOID: NO NVB: 15NOV25 NVA: 15DEC25
